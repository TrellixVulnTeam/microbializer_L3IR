Introduction

Significant technological advances in the last decade have enabled the use of large-scale sequencing, mining, and analysis of bacterial species data in an unprecedented resolution. Such large-scale analyses have contributed to the comprehensive characterization of complex microbial dynamics within a microbiome [1] and among different strains during disease outbreaks [2]. Today, such an analysis begins with preliminary steps including genome annotation and orthologs detection, sequence alignment, and accurate phylogenetic tree reconstruction. These steps as well as additional analysis-specific computations require the use of multiple bioinformatics tools and various ad-hoc programming scripts, making the entire process cumbersome, tedious, and prone to errors due to manual handling. This has prompted laboratories to implement their own in-house analysis pipelines, and thus different analysis applications have begun to emerge [3, 4]. These applications require specific working environments, i.e., operating system, computation power, i.e., multi-cores machines, and more than basic technological skills, such as installations and running.

What is M1CR0B1AL1Z3R?

M1CR0B1AL1Z3R (pronounced: microbializer) was developed in order to facilitate large scale microbial genomics analyses. Our goal was to make M1CR0B1AL1Z3R easily accessible to the scientific community and thus to allow the abovementioned computational challenges of analyzing dozens of bacterial genomes simultaneously. Such computations are highly demanding, and therefore, M1CR0B1AL1Z3R runs background computational processes in parallel on a high-performance computer cluster. Notably, due to the high computational demands of these analyses, they cannot run on a simple desktop computer. To this end, M1CR0B1AL1Z3R enables the research community to analyze massive genomic data that previously necessitated expensive computational infrastructure and expert knowledge. Both visual and textual results that are ready for publication or further analysis are given as an output.

Input

The input for the webserver is a zipped folder (either .zip or .tar.gz) in which each file is a fasta nucleic acid (.fna) file containing genomic sequences of a different species (whether fully sequenced or as contigs).



Output and methodology

M1CR0B1AL1Z3R implements the following algorithmic steps:
  • ORF extraction from all genomes using Prodigal [5].
  • Homologous gene detection (all against all) and clustering using MCL [6, 7]. This results in a presence-absence phyletic database (a detailed table in which row i is an orthologous group, column j is a bacterial genome and the i,j entry contains the member of orthologous family i from genome j, if present.
  • For each orthologous group, M1CR0B1AL1Z3R provides a MAFFT-LINSI [8] multiple sequence alignment of all members (the unaligned sequences are provided as well).
  • M1CR0B1AL1Z3R compute the core-genome (those genes that are shared across all bacterial genomes) and provide the core-genome alignment, based on which, the bacterial species tree is often inferred. In case where there are duplications (i.e., identical core-genomes), a core-genome alignment containing only unique sequences is provided as well.
  • Based on the core-genome alignment a maximum-likelihood based phylogenetic tree is reconstructed using RAxML [9] and visualised by PhyD3 [10].
  • Several statistics regarding the pan genome (the entire set of genomes analyzed) are also provided, e.g., the GC content, the distribution of the number of ORFs across the analyzed genomes, and the distribution of the sizes of the orthologous groups.


M1CR0B1AL1Z3R provides the following results:
  • A curated table of orthologous sets;
  • A fasta file for each orthologous set containing the UNALIGNED sequences of the set members;
  • A fasta file for each orthologous set containing the ALIGNED sequences of the set members;
  • A histogram* of the group sizes frequency.
  • A violin plot* of the ORF counts per genome.
  • A violin plot* of the ORF GC-content per genome.
  • A fasta file containing the inferred core-genome of the species in the dataset;
  • A reliable species phylogeny tree* as an interactive figure;
*Raw data is provided as well
