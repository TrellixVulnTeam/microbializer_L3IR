Why was a species tree not included in my analysis results?

There are several cases in which a phylogenetic tree is absent from the results. This can reflect datasets with less than four different genomic sequences. In this case there is no tree because unrooted trees for less than four sequences is trivial (there is only a single tree). Another possibility is that there are no core genes. The tree is built based on the core proteome, and if there are no core genes (genes shared by all samples), there is no data to build the tree. A way around this is to change the definition of core genes in the advanced options. Sometimes there are more than three sequences, there are core genes and still a tree is missing. This can reflect genomic sequences that are identical. RAxML reduces the input alignment to unique sequences, i.e., it removes duplicates.


Why does the homologs inference step take so long?

When searching for homologs, each gene in each species is queried against all other genes in all other taxa, making it the most computationally intensive step. Even though M1CR0B1AL1Z3R uses the fastest algorithm that is currently available for this task and despite the fact it uses parallelization for this computational step, for a dataset with ~150 genomes, this step might take even 10 hours.
